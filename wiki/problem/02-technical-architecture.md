## 2.1 插件 API 兼容性策略

**决策**：选择 **B 方案 - 兼容 + 扩展**

**兼容性实现方式**：
- 兼容 uTools 核心 API（`window.utools`）
- 扩展 Corelia 特有 API（如 WASM 补丁调用、云端同步等）

**API 兼容性策略**：
```
┌─────────────────────────────────────────────────────────────┐
│                    插件 API 层                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │              window.utools（兼容）                   │  │
│   │   dbStorage / clipboard / shell / createBrowserWindow │  │
│   └─────────────────────────────────────────────────────┘  │
│                            │                               │
│                            ▼                               │
│   ┌─────────────────────────────────────────────────────┐  │
│   │          window.corelia（扩展）                     │  │
│   │   wasm.patch() / sync.config() / ai.chat()          │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**已确定** ✅：
- [x] 兼容性实现方式 → **兼容 + 扩展**
- [x] 是否需要同时兼容 ZTools API？ → **否，MVP 只兼容 uTools**
- [ ] API 兼容性测试方案？ → 待补充

## 2.2 QuickJS 与 Webview 集成细节

**决策**：QuickJS 和 Webview **统一架构设计**，共享通信机制

### 统一通信架构

```
┌─────────────────────────────────────────────────────────────┐
│                    插件运行时层                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────┐         ┌─────────────────────┐    │
│   │   QuickJS 插件   │         │    Webview 插件      │    │
│   │                 │         │                      │    │
│   │  • JSON 配置驱动 │         │  • Vue/React/Svelte │    │
│   │  • 7ms 响应     │         │  • iframe 嵌入       │    │
│   │                 │         │                      │    │
│   └────────┬────────┘         └──────────┬───────────┘    │
│            │                              │                │
│            └──────────┬───────────────────┘                │
│                       ▼                                     │
│            ┌─────────────────────┐                         │
│            │   Plugin Bridge     │                         │
│            │   (Tauri Event)     │                         │
│            └──────────┬──────────┘                         │
│                       │                                     │
│                       ▼                                     │
│   ┌─────────────────────────────────────────────────────┐  │
│   │              Rust Core (WASM 补丁 / System API)     │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### QuickJS 集成细节

| 问题 | 决策 | 说明 |
|------|------|------|
| 通信机制 | **Tauri Event** | 与 Webview 插件统一 |
| 全局对象 | **`window.utools`** | 兼容 uTools API |
| VM 实例 | **单例** | 降低复杂度 |
| 代码加载 | **本地文件** | `plugins/{plugin_id}/index.js` |

### Webview 集成细节

| 问题 | 决策 | 说明 |
|------|------|------|
| 隔离方式 | **iframe（MVP）** | 后续可扩展多窗口 |
| 通信方式 | **Tauri Event** | 与 QuickJS 插件统一 |
| 生命周期 | **按需创建/销毁** | 插件退出时销毁 iframe |

### 统一 API 桥接

```typescript
// 插件统一调用方式
window.utools.dbStorage.setItem('key', 'value');  // QuickJS & Webview

// Corelia 扩展 API
window.corelia = {
  wasm: {
    call(patch: string, api: string, ...args: any[]): Promise<any>;
  },
  sync: {
    config(): Promise<SyncConfig>;
  },
  ai: {
    chat(prompt: string, stream?: boolean): Promise<string>;
  }
};
```

**已确定** ✅：
- [x] QuickJS 与 Webview 统一通信机制 → **Tauri Event**
- [x] QuickJS 全局对象 → **`window.utools`**
- [x] Webview 隔离方式 → **iframe（可扩展多窗口）**

## 2.4 WASM 补丁技术选型

**决策**：按建议方案

| 问题 | 决策 | 说明 |
|------|------|------|
| 工具链 | **wasm-pack** | 成熟稳定 |
| FFI 方案 | **wasm-bindgen** | 与 wasm-pack 配套 |
| MVP 复杂度 | **优先社区成熟库** | 降低风险 |

**WASM 补丁构建流程**：
```
Rust Code
    │
    ▼
wasm-pack build --target web
    │
    ▼
.wasm 文件 + js glue
    │
    ▼
WASM 补丁加载器
    │
    ▼
插件通过 window.corelia.wasm.call() 调用
```

**已确定** ✅：
- [x] 工具链 → **wasm-pack**
- [x] FFI → **wasm-bindgen**
- [x] MVP 策略 → **社区库优先**

# 功能需求规格说明书 (SRS)

## 1. 项目概述

### 1.1 项目背景

Corelia 是一款面向桌面端的效率工具，定位为类似 uTools、Listary、Alfred 的**快速启动器 + 插件平台**。用户通过全局快捷键唤起搜索框，快速搜索并执行系统功能、插件功能或第三方工具。

### 1.2 项目目标

构建一个**轻量、安全、可扩展**的桌面效率工具平台，具备：

- **快速唤起**：全局快捷键毫秒级响应
- **模糊搜索**：输入即匹配，支持插件和系统功能
- **插件生态**：支持用户自行开发、加载第三方插件
- **系统集成**：深度整合文件系统、剪贴板、Shell 等系统能力

### 1.3 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 框架 | Tauri 2.x | 轻量、安全、原生体验 |
| 前端 | Svelte 5 + SvelteKit | 响应式 UI、高性能 |
| 后端 | Rust | 系统级能力、高性能 |
| 构建 | Vite + Bun | 快速开发、热更新 |

#### 2.2.3 插件懒加载模式

当用户安装大量插件时，启动时加载所有插件会导致性能问题。引入懒加载模式，只在用户真正使用时才完整加载插件代码和资源。

| 需求编号 | 功能描述 | 优先级 | 备注 |
|----------|----------|--------|------|
| F-044 | 插件基础信息预加载 | P0 | 启动时仅加载插件名称、图标、前缀等元数据 |
| F-045 | 插件懒加载触发条件 | P0 | 用户输入匹配到插件前缀时触发完整加载 |
| F-046 | 插件代码按需加载 | P0 | 仅加载插件的入口 JS/CSS/HTML 文件 |
| F-047 | 插件缓存机制 | P1 | 已加载的插件缓存在内存中，避免重复加载 |
| F-048 | 插件预加载策略 | P2 | 根据用户使用频率智能预加载常用插件 |
| F-049 | 懒加载状态管理 | P1 | 跟踪插件的加载/卸载/缓存状态 |
| F-049a | 插件内存管理 | P1 | 当插件数量过多时，自动释放不常用插件的内存 |

**懒加载流程设计**：

```mermaid

flowchart TD
    A[应用启动] --> B[扫描 plugins/ 目录]
    B --> C[仅解析 plugin.json 元数据]
    C --> D[构建轻量级插件索引]
    D --> E[用户输入搜索关键词]
    E --> F{匹配到插件前缀?}
    F -->|是| G[触发插件懒加载]
    F -->|否| H[显示系统功能/历史记录]
    G --> I{插件已缓存?}
    I -->|是| J[直接使用缓存]
    I -->|否| K[动态加载插件资源]
    K --> L[执行插件初始化代码]
    L --> M[缓存插件实例]
    M --> N[渲染插件结果]
    J --> N
    N --> O[用户选择结果]
    O --> P[调用插件功能]

```

**懒加载状态机**：

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| `MetaLoaded` | 仅元数据已加载 | 应用启动 |
| `Loading` | 正在加载完整插件 | 用户输入匹配到前缀 |
| `Ready` | 插件已就绪 | 资源加载完成 |
| `Cached` | 插件已缓存 | 加载完成后自动进入 |
| `Unloaded` | 插件已卸载 | 内存压力或手动卸载 |

##### 2.2.6.1 QuickJS 插件

适用于简单功能的轻量级插件，通过 JSON 配置驱动。

| 特性 | 说明 |
|------|------|
| **加载方式** | 懒加载，首次匹配前缀时实例化 QuickJS VM |
| **响应时间** | ~7ms |
| **内存占用** | ~1-2MB/实例 |
| **适用场景** | 简单列表、命令聚合、数据转换 |

**QuickJS 插件结构**：

```
my-quickjs-plugin/
├── plugin.json
└── index.js       # 可选，扩展默认行为
```

**plugin.json 示例**：

```json
{
  "name": "快捷命令",
  "version": "1.0.0",
  "type": "quickjs",
  "logo": "logo.png",
  "patches": ["crypto"],
  "features": [
    {
      "code": "cmd",
      "label": "命令列表",
      "type": "list",
      "items": [
        { "label": "打开计算器", "action": "calc" },
        { "label": "打开记事本", "action": "notepad" }
      ]
    }
  ]
}
```

##### 2.2.6.3 WASM 补丁

提供高性能原生能力，供 QuickJS 和 Webview 插件调用。

| 特性 | 说明 |
|------|------|
| **加载方式** | 插件依赖时自动加载，支持缓存复用 |
| **响应时间** | ~200-400ms（首次编译），<10ms（调用）|
| **内存占用** | ~5-20MB/实例 |
| **适用场景** | AI API、加密算法、图像处理 |

**WASM 补丁结构**：

```
patches/
├── ai-v1/
│   ├── patch.json
│   ├── ai.wasm
│   └── api.d.ts        # TypeScript 类型定义（可选）
└── crypto-v1/
    ├── patch.json
    └── crypto.wasm
```

**patch.json 示例**：

```json
{
  "name": "ai-v1",
  "version": "1.0.0",
  "description": "AI 模型集成补丁",
  "apis": ["aiModels", "aiChat", "aiEmbedding", "aiStreamChat"],
  "size": "2.5MB"
}
```

##### 2.2.6.4 插件与补丁依赖关系

```mermaid

flowchart TD
    A[用户输入 "ai"] --> B[匹配到 AI助手 插件]
    B --> C{plugin.json 解析}
    C --> D{有 patches?}
    D -->|无| E[直接初始化插件]
    D -->|有| F{补丁已缓存?}
    F -->|是| G[版本兼容?]
    F -->|否| H[加载 WASM 补丁]
    H --> I[AOT 编译]
    I --> J[实例化并注册 API]
    J --> K[添加到缓存池]
    K --> G
    G -->|是| L[复用缓存]
    G -->|否| H
    L --> E
```

**插件依赖声明示例**：

```json
{
  "name": "AI助手",
  "type": "webview",
  "patches": ["ai-v1"],
  "main": "index.html"
}
```

## 3. 非功能需求

### 3.1 性能需求

| 需求编号 | 指标 | 目标值 |
|----------|------|--------|
| NF-001 | 冷启动时间 | < 2 秒 |
| NF-002 | 窗口唤起延迟 | < 100ms |

| NF-004 | 内存占用（空闲） | < 100MB |
| NF-005 | 插件加载时间 | QuickJS < 10ms, Webview < 500ms |

### 3.2 性能分析

#### 3.2.1 整体性能模型

```
启动时间: O(扫描) + O(解析JSON)       ← 极轻量
搜索响应: O(n) 模糊匹配                ← 毫秒级
插件加载: O(插件类型) + O(依赖WASM)    ← 按需
内存占用: 固定开销 + 动态插件内存       ← 可控
```

#### 3.2.2 启动性能

| 阶段 | 时间复杂度 | 实际耗时（100个插件）|
|------|------------|---------------------|
| 扫描目录 | O(n) | ~50ms |
| 解析 JSON | O(n) | ~10ms |
| 构建搜索索引 | O(n log n) | ~20ms |
| **总计** | | **~80ms** |

**启动时不加载**：
- ❌ 不实例化 QuickJS VM
- ❌ 不创建 Webview
- ❌ 不加载 WASM
- ❌ 不加载插件图标（仅在搜索结果展示时按需加载）

#### 3.2.3 插件加载性能

| 插件类型 | 首次加载 | 缓存复用 | 内存占用 |
|----------|----------|----------|----------|
| **QuickJS** | ~7ms | <1ms | ~1-2MB |
| **Webview** | ~200-350ms | <50ms | ~30-50MB |

## 4. 数据流设计

### 4.1 核心数据流

```mermaid

sequenceDiagram
    participant User as 用户
    participant Frontend as 前端 (Svelte)
    participant Tauri as Tauri Core
    participant Rust as Rust 后端
    participant Plugin as 插件系统
    participant OS as 操作系统

    User->>Frontend: 输入搜索关键词
    Frontend->>Plugin: 模糊匹配插件列表
    Plugin->>Frontend: 返回匹配结果
    Frontend->>Frontend: 渲染搜索结果列表
    User->>Frontend: 选择插件/功能
    Frontend->>Tauri: 调用 Command
    Tauri->>Rust: 路由到具体处理函数
    Rust->>OS: 调用系统 API (fs/clipboard/shell)
    OS-->>Rust: 返回执行结果
    Rust-->>Tauri: 返回结果
    Tauri-->>Frontend: 事件通知
    Frontend->>Frontend: 更新 UI / 关闭窗口

```

### 4.2 插件加载流程

```mermaid

flowchart TD
    A[应用启动] --> B[扫描 plugins/ 目录]

    C -->|读取 plugin.json| D[解析元数据]
    D --> E[验证插件格式]
    E --> F[注册到插件管理器]
    F --> G[加载插件入口文件]
    G --> H[渲染到搜索索引]
```

## 7. 术语表

| 术语 | 定义 |
|------|------|
| **效率工具** | 帮助用户快速完成日常任务的桌面应用 |
| **全局快捷键** | 在应用未获得焦点时也能响应的键盘快捷键 |
| **插件系统** | 可扩展的功能模块，允许第三方开发者添加新功能 |
| **沙箱隔离** | 将插件运行在受限环境中，防止影响主程序 |
| **模糊匹配** | 不完全匹配也能找到结果的搜索算法 |
| **Webview** | 嵌入在原生应用中的网页渲染组件 |
| **QuickJS** | 轻量级 JavaScript 引擎，用于运行简单插件 |
| **WASM 补丁** | WebAssembly 编译的原生能力模块，提供高性能功能 |
| **三层插件架构** | QuickJS 插件 + Webview 插件 + WASM 补丁的分层设计 |
| **懒加载** | 延迟加载插件代码到首次使用时再加载的模式 |
| **AOT 编译** | Ahead-of-Time 编译，预先将 WASM 编译为本地代码 |

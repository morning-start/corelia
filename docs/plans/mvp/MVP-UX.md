# MVP-UX 用户体验优化计划

## 概述

本计划用于指导 MVP 用户体验优化阶段，包括 Onboarding、快捷键、搜索体验、主题等用户体验相关的优化。

---

## 目标与验收标准

### 目标

优化 Corelia 的用户体验，包括：

1. **首次使用引导（Onboarding）**：4 步引导流程
2. **快捷键优化**：自定义、冲突检测、提示
3. **搜索体验优化**：分词、高亮、历史
4. **结果展示优化**：分组、图标、描述
5. **窗口行为优化**：动画、过渡、多显示器

### 验收标准

| 序号 | 验收标准 | 验证方法 | 通过标准 |
|------|---------|---------|---------|
| 1 | Onboarding 4 步引导正常 | 手动测试 | 引导正常完成 |
| 2 | 快捷键自定义生效 | 手动测试 | 自定义键位正常 |
| 3 | 快捷键冲突提示正常 | 手动测试 | 冲突时提示 |
| 4 | 搜索响应 < 50ms | 性能测试 | 1000 条数据下 |
| 5 | 主题切换正常 | 手动测试 | 深/浅/跟随正常 |
| 6 | 结果分组显示正常 | 手动测试 | 分组正确 |

### 详细任务说明

#### UX-01 Onboarding 引导

**任务描述**：实现首次使用引导流程

**引导流程（4 步）**：

| 步骤 | 内容 | 时长 |
|------|------|------|
| 1 | 设置快捷键 | 30 秒 |
| 2 | 介绍核心插件（快捷命令、剪贴板增强、文件搜索） | 1 分钟 |
| 3 | 展示其他 4 个插件入口 | 15 秒 |
| 4 | 完成，进入主界面 | - |

**执行步骤**：
1. 创建 Onboarding 组件
2. 实现步骤导航
3. 实现快捷键设置
4. 实现核心插件介绍
5. 实现完成跳转
6. 实现首次使用标记

**组件**：`src/lib/components/Onboarding.svelte`

#### UX-02 快捷键冲突检测

**任务描述**：实现快捷键冲突检测和提示

**执行步骤**：
1. 实现冲突检测逻辑
2. 实现冲突提示 UI
3. 实现冲突解决方案
4. 实现冲突历史记录

**冲突处理策略**：
- 检测时机：首次使用时
- 冲突提示：提示用户手动修改
- 不强制覆盖系统快捷键

#### UX-03 搜索结果高亮

**任务描述**：实现搜索结果的高亮显示

**执行步骤**：
1. 实现匹配字符高亮
2. 实现拼音匹配高亮
3. 实现高亮样式定制

**高亮样式**：
- 匹配字符使用 `<mark>` 标签
- 高亮颜色使用主题变量 `--color-accent`

#### UX-04 搜索历史

**任务描述**：实现搜索历史记录和展示

**执行步骤**：
1. 实现历史记录存储
2. 实现历史记录展示
3. 实现历史记录清除
4. 实现历史记录搜索

**历史记录规则**：
- 最多保存 100 条
- 按使用频率排序
- 支持一键清除

#### UX-05 搜索结果分组

**任务描述**：实现搜索结果的分组展示

**执行步骤**：
1. 定义分组规则
2. 实现分组展示 UI
3. 实现分组切换
4. 实现分组排序

**默认分组**：
- 插件（按插件名称）
- 系统功能
- 历史记录

#### UX-06 窗口动画效果

**任务描述**：实现窗口的动画效果

**执行步骤**：
1. 实现窗口出现动画
2. 实现窗口隐藏动画
3. 实现结果项动画
4. 实现主题切换动画

**动画设计**：
- 窗口出现：fadeIn + scale
- 窗口隐藏：fadeOut + scale
- 动画时长：150-200ms

#### UX-07 多显示器支持

**任务描述**：实现多显示器环境下的窗口显示

**执行步骤**：
1. 获取显示器信息
2. 确定窗口显示位置
3. 实现位置记忆
4. 实现显示器热插拔处理

**显示策略**：
- 默认在当前激活显示器显示
- 记住用户偏好位置
- 显示器变更时自动调整

#### UX-08 国际化准备

**任务描述**：为国际化做准备

**执行步骤**：
1. 提取 UI 文本
2. 创建国际化文件结构
3. 集成 i18n 框架
4. 支持中文和英文

**文件结构**：
```
src/
└── locales/
    ├── zh-CN.json
    └── en-US.json
```

#### UX-09 用户体验测试

**任务描述**：进行用户体验测试

**测试内容**：
1. Onboarding 流程测试
2. 快捷键使用测试
3. 搜索体验测试
4. 主题切换测试
5. 整体流畅度评估

**测试方法**：
- 内部测试
- 收集反馈
- 迭代优化

## 用户体验设计规范

```css

  --font-family: "Segoe UI", system-ui, sans-serif;
  --font-size-small: 12px;
  --font-size-normal: 14px;
  --font-size-large: 16px;
  --font-size-xlarge: 20px;
}
```

### 颜色规范

```css
:root {
  --color-bg: #1a1a1a;
  --color-text: #ffffff;
  --color-text-secondary: #888888;
  --color-accent: #0078d4;
  --color-accent-hover: #1084d8;
  --color-border: #333333;
  --color-success: #4caf50;
  --color-warning: #ff9800;
```

```css

  --spacing-xs: 4px;

  --spacing-xl: 24px;
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 12px;
```

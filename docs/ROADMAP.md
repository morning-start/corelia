# Corelia 项目路线图 (ROADMAP)

> 本文档描述 Corelia 的长期发展规划与阶段性里程碑。
> 更新周期：每季度评审一次。

---

## 阶段一：MVP 核心框架（当前阶段 🔄）

**目标窗口**：2026 Q2 - 2026 Q3  
**核心目标**：完成桌面快速启动器的最小可用产品，验证技术架构可行性。

### 交付标准

| 维度 | 目标 |
|------|------|
| 功能 | 全局唤起 + 模糊搜索 + 插件系统基础 + 4个示例插件 |
| 性能 | 窗口唤起 < 100ms，搜索响应 < 50ms |
| 稳定性 | 插件崩溃隔离，主程序不受影响 |
| 文档 | API 文档 + 插件开发指南 + 架构文档 |

---

## 阶段二：插件生态孵化（规划中 📋）

**目标窗口**：2026 Q3 - 2026 Q4  
**核心目标**：补齐 MVP 约定的 8 个核心插件，建立插件开发闭环。

### 关键任务

- **核心插件完善**：计算器、文件搜索、URL 工具、剪贴板增强等
- **WASM Patch 扩展**：crypto 补丁稳定运行，AI 补丁预研
- **插件管理界面**：安装 / 卸载 / 启用 / 禁用可视化操作
- **插件开发 SDK**：TypeScript 类型定义 + 开发模板

### 交付标准

- 8 个官方插件全部可用
- 第三方开发者可基于文档独立开发插件
- 插件热重载功能稳定

---

## 阶段三：Beta 发布与打磨（规划中 📋）

**目标窗口**：2026 Q4 - 2027 Q1  
**核心目标**：面向早期用户发布 Beta 版本，收集反馈并迭代。

### 关键任务

- **跨平台适配**：macOS (Intel + Apple Silicon) 初步支持
- **分发机制**：GitHub Release + .zip 拖拽安装
- **首次使用引导**：4 步 Onboarding 流程
- **设置面板完善**：主题 / 快捷键 / 行为偏好全覆盖
- **性能优化**：启动时间 < 1 秒，内存占用 < 100MB

### 交付标准

- Windows / macOS 双平台运行
- 无阻塞性 Bug，核心体验流畅
- 社区渠道（GitHub Discussions）建立

---

## 阶段四：v1.0 正式发布（规划中 📋）

**目标窗口**：2027 Q2  
**核心目标**：发布稳定版本，建立可持续的插件生态。

### 关键任务

- **插件市场**：内置插件浏览与一键安装界面
- **安全机制**：插件签名验证 + 沙箱权限分级
- **Linux 适配**：Ubuntu 22.04+ / Fedora 39+
- **企业级需求**：开机自启动、稳定性、无障碍支持

### 交付标准

- 三平台（Windows / macOS / Linux）稳定运行
- 插件市场有 ≥ 20 个可用插件
- 用户文档完整，开发文档成熟

---

## 阶段五：v2.0 生态扩展（长期 🎯）

**目标窗口**：2027 H2 - 2028  
**核心目标**：从工具平台向效率生态演进。

### 方向探索

| 方向 | 说明 |
|------|------|
| AI 助手集成 | 基于 WASM 补丁的智能推荐与对话能力 |
| 工作流自动化 | 插件间联动，自定义工作流编排 |
| 云端同步 | 配置与插件数据跨设备同步 |
| 移动端延伸 | 配套移动端 App（远程唤起 / 剪贴板同步）|

> Corelia 安全风险与缓解措施

## 安全架构

### 安全设计原则

1. **最小权限** - 插件仅能访问声明的 API
2. **沙箱隔离** - 插件运行在隔离环境
3. **用户授权** - 敏感操作需用户确认
4. **数据加密** - 敏感数据加密存储

## 安全风险点

### 1. QuickJS `unsafe impl Send/Sync`

**位置**: `src-tauri/src/plugins/quickjs_runtime.rs`

**问题代码**:
```rust
unsafe impl Send for VmInstance {}
unsafe impl Sync for VmInstance {}
```

**风险评估**:
- **当前风险**: 低
- **未来风险**: 中

**当前假设**:
- Tauri Commands 在主线程执行
- VM 实例不会跨线程访问

**缓解措施**:
1. 增加 `#[cfg(debug_assertions)]` 断言检测跨线程访问
2. 考虑用 `Mutex` 替代 `RefCell`
3. 详细文档化此设计决策

---

### 2. 插件数据隔离

**风险**: 插件可能访问其他插件的数据

**缓解措施**:
- 每个插件独立存储空间
- 数据路径隔离
- API 层权限检查

---

### 3. 网络请求安全

**风险**: 插件可能发起恶意网络请求

**缓解措施**:
- 网络请求需用户授权（P2）
- 域名白名单机制
- 请求日志记录

---

### 4. 文件系统访问

**风险**: 插件可能访问敏感文件

**缓解措施**:
- 通过 Tauri allowlist 控制
- 限制访问路径
- 用户确认敏感操作

## 数据安全

### 敏感数据处理

| 数据类型 | 存储方式 | 访问控制 |
|----------|----------|----------|
| 用户配置 | 明文 JSON | 文件系统权限 |
| 插件数据 | 明文 JSON | 插件隔离 |
| API 密钥 | 加密存储 | 用户授权 |
| 历史记录 | 明文 JSON | 本地访问 |

### 数据清理

- 插件卸载时清理数据
- 提供数据导出功能
- 支持用户手动清理

## 安全最佳实践

### 开发者指南

1. **不要硬编码密钥** - 使用环境变量或安全存储
2. **验证所有输入** - 特别是来自插件的数据
3. **最小权限原则** - 仅请求必要的 API 权限
4. **安全日志** - 记录敏感操作，不记录敏感数据

### 代码审查要点

1. 检查 `unsafe` 代码块
2. 验证用户输入处理
3. 检查文件路径处理
4. 审查网络请求代码

---

## 安全事件响应

### 发现漏洞时

1. **评估影响** - 确定漏洞严重程度
2. **修复漏洞** - 优先处理高危漏洞
3. **发布更新** - 及时发布修复版本
4. **通知用户** - 告知风险和解决方案

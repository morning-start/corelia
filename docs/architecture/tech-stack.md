# 技术栈分析

> Corelia 技术选型与评估

---

## 技术栈总览

| 层级 | 技术选型 | 版本 | 评估 |
|------|----------|------|------|
| **桌面框架** | Tauri | 2.x | ✅ 优秀 |
| **前端框架** | Svelte | 5.x | ✅ 优秀 |
| **前端应用框架** | SvelteKit | 2.x | ✅ 优秀 |
| **后端语言** | Rust | 1.75+ | ✅ 优秀 |

---

## 前端技术栈

### Svelte 5

**选型理由**:
- 编译时框架，无运行时开销
- 5.x 版本引入 Runes，响应式更精细
- 学习曲线平缓，开发效率高

**注意事项**:
- 优先使用 Runes 语法 (`$state`, `$derived`)
- 部分旧代码仍用 `writable`，需逐步迁移

### SvelteKit

**选型理由**:
- 官方应用框架，与 Svelte 深度集成
- 支持静态站点生成 (adapter-static)
- 文件系统路由，开发体验好

### TypeScript

**选型理由**:
- 类型安全，减少运行时错误
- IDE 支持完善
- 团队熟悉

**配置**:
- 严格模式开启
- 版本锁定 ~5.6.2

---

## 后端技术栈

### Rust

**选型理由**:
- 内存安全，无 GC
- 高性能，适合系统级开发
- 与 Tauri 深度集成

**注意事项**:
- 学习曲线较陡
- 编译时间较长

### Tauri 2.x

**选型理由**:
- 比 Electron 更轻量
- 原生安全模型
- 2.x 版本 API 更完善

**注意事项**:
- 2.x 与 1.x API 有变化
- 关注官方迁移指南

### QuickJS (rquickjs)

**选型理由**:
- 轻量级 JS 引擎
- 适合沙箱化插件执行
- 启动快速，内存占用低

**注意事项**:
- 不支持所有 ES6+ 特性
- Promise 支持有限
- 关注 rquickjs 上游更新

---

## 构建工具

### Vite

**选型理由**:
- 开发服务器快速
- HMR 体验好
- 与 SvelteKit 深度集成

### Bun

**选型理由**:
- 比 npm/yarn 更快
- 内置 TypeScript 支持
- 一体化工具链

**注意事项**:
- Windows 支持需注意执行策略
- 部分包可能兼容性问题

## 技术栈对比

### 与 Electron (uTools) 对比

| 维度 | Corelia (Tauri) | uTools (Electron) |
|------|-----------------|-------------------|
| 安装包大小 | ~15MB | ~100MB+ |
| 内存占用 | ~40-100MB | ~150MB+ |
| 启动速度 | 快 | 较慢 |
| 生态成熟度 | 发展中 | 成熟 |

### 与 Delphi+CEF (ZTools) 对比

| 维度 | Corelia | ZTools |
|------|---------|--------|
| 开发语言 | Rust + TypeScript | Delphi + JS |
| 跨平台 | 较易 | 较难 |
| 学习曲线 | 中等 | 较陡 |

## 技术债务

### 当前需要关注

1. **Svelte 5 Runes 迁移** - 部分代码仍用旧语法
2. **Bun 兼容性** - 注意 Windows 执行策略
3. **QuickJS Promise 支持** - 关注上游更新

### MVP 后优化

1. **统一包管理** - 完全迁移到 Bun
2. **模块化拆分** - api_bridge.rs、loader.rs 文件过大

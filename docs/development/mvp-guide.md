# MVP 开发指南

> Corelia MVP 阶段核心开发指南

---

## MVP 目标

| 目标 | 衡量标准 |
|------|----------|
| 功能完整 | 7个QuickJS插件 + 1个Webview插件可工作 |
| 性能达标 | 冷启动<2s，搜索响应<50ms |
| 架构解耦 | 模块边界清晰，易于扩展 |
| 可交付 | 文档完整，安装流程顺畅 |

---

## 开发路线图

| 阶段 | 时间 | 主要任务 |
|------|------|----------|
| **Phase 1** | Week 1 | 架构优化与Bug修复 |
| **Phase 2** | Week 2-3 | MVP 插件开发 |
| **Phase 3** | Week 4 | Webview插件 + 测试 |
| **Phase 4** | Week 5 | 文档 + 发布准备 |

## Phase 1: 架构优化

### 必须完成的优化

| 任务 | 优先级 | 工时 |
|------|--------|------|
| 修复 22 个 Svelte 警告 | P0 | 2h |
| 插件搜索并行化 | P0 | 2h |
| 移除前端 VM 缓存 | P0 | 4h |
| WASM 轮询退避策略 | P1 | 2h |
| 拼音搜索短路优化 | P1 | 1h |

### 关键任务详解

#### 移除前端 VM 缓存

**问题**: 前端 `src/lib/plugins/service.ts` 有 VM 缓存，与后端重复

**方案**: 移除所有 `vmCache` 相关代码，改为纯代理模式

**涉及文件**:
- `src/lib/plugins/service.ts`
- `src/lib/components/PluginManager.svelte`

## 前后端职责边界

### 前端负责

- ✅ UI 渲染
- ✅ 状态管理
- ✅ 用户输入处理
- ✅ 搜索逻辑
- ✅ 错误展示

### 前端禁止

- ❌ 管理 VM
- ❌ 持久化数据
- ❌ 直接文件系统操作

### 后端负责

- ✅ 系统能力封装
- ✅ 插件生命周期
- ✅ 配置持久化
- ✅ 资源池管理

---

## MVP "做"与"不做"

### ✅ 应该做

1. 完成 7+1 个插件
2. 修复 P0-P1 级问题
3. 统一文档结构
4. 保证核心功能可用

### ❌ 不要做

1. 过度优化（P2及以下）
2. 功能镀金
3. 大规模重构
4. 跨平台适配
5. 完整测试覆盖
